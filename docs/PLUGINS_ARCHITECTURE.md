# Архитектура Плагинов (Smart Intercom Hub)

> Актуальный статус реализации: **Prototype v2.0**

Система Smart Intercom Hub расширяется за счёт двух независимых механизмов.

---

## 1. Internal Plugins — Внутренние плагины (Реализовано)

Лёгкие TypeScript-скрипты, выполняемые **внутри Node.js-процесса ядра**. Идеальны для интеграций, требующих доступа к данным карточек людей и логике открытия двери.

**Технический справочник:** [`docs/PLUGIN_API.md`](./PLUGIN_API.md)  
**Типы API:** [`plugins/plugin-api.d.ts`](../plugins/plugin-api.d.ts)

### Реализованные возможности

| Функция | Статус |
|---|---|
| Изолированное хранилище (`api.storage`) | ✅ |
| HTTP-эндпоинты (`api.router`) | ✅ |
| Регистрация Capability + UI-переключатели | ✅ |
| Хук загрузки карточек (`onPersonLoad`) | ✅ |
| Хук решения об открытии (`onEvaluateAutoOpen`) | ✅ |
| Расширение UI карточек (`uiExtensions`) | ✅ |
| Автосканирование папки `/plugins` | 🔄 В планах |
| Hot Reload плагинов | 🔄 В планах |

### Текущие плагины

- Нет активных плагинов по умолчанию.

## 2. External Modules — Внешние модули (Реализовано)

Внешние скрипты могут регистрироваться в системе динамически через HTTP REST API и WebSockets. Ядро выступает универсальным брокером, а модули (например, Python скрипт для Face ID `dom-ru-eye`) интегрируются по факту запуска.

**Документация:** [`README.md`](../README.md#external-modules)

### Архитектура подключения

Модуль может получать события от Ядра тремя способами. Выбор способа зависит от среды выполнения модуля (Serverless, локальный скрипт и т.д.):

1. **WebSockets (По умолчанию)**
   - Модуль (как постоянно запущенный процесс) подключается к пространству имен `/modules`: `wss://ВАШ_ДОМЕН/modules?token=ТОКЕН`.
   - Идеально подходит для скриптов (Python, Node.js), работающих в фоне (например, `face-id-module`).
   - По этому каналу ядро сообщает статус "В сети" для веб-интерфейса.

2. **Webhooks (HTTP POST)**
   - Вы можете указать `Webhook URL` в настройках модуля через UI или зарегистрировать его через `POST /api/modules/actions/connection`.
   - Ядро будет отправлять HTTP POST запросы на этот URL при каждом событии.
   - Идеально для бессерверных сред (**Serverless**, например, Vercel) или no-code платформ (Make.com, n8n), которые не могут удерживать постоянное WebSocket соединение.

3. **Long Polling (HTTP GET)**
   - Модуль делает `GET /api/modules/actions/poll?token=ТОКЕН`.
   - Ядро "замораживает" ответ на срок до 30 секунд. Как только происходит событие, Ядро мгновенно возвращает его.
   - Подходит для скриптов за NAT/роутерами, у которых нет белого IP для получения Webhooks.

### Основной функционал

1. **Токены:** Пользователь генерирует API-токен в интерфейсе → «Внешние модули».
2. **События:** В зависимости от типа подключения, Ядро рассылает события: `incoming_call` (входящий вызов) и `door_opened`.
3. **Capabilities (Возможности):** Модуль регистрирует себя через `POST /api/modules/actions/capabilities`, указывая `capability` (например, `FACE_RECOGNITION`), что динамически включает тумблеры и настройки в карточках пользователей.
4. **Storage API (Загрузка медиа):** Модуль может запросить медиа-файлы (фото) пользователя через `GET /api/modules/storage/:moduleId/:personId`. Фото загружаются прямо в ядро пользователем через веб-интерфейс.
5. **Stream API (Видеопоток):** При звонке модуль может получить прямую HLS/RTSP ссылку на камеру через `GET /api/modules/actions/stream/:deviceId`.
6. **Open API (Управление дверью):** Если модуль принял решение открыть дверь, он делает `POST /api/modules/actions/open` (авторизация по Bearer-токену) с передачей `capability` и `personId`. Ядро запишет это событие в лог и откроет дверь.

## 3. Roadmap — Отложенные идеи

### 3.1. Голосовые оповещения Алисы (Proactive TTS)
**Идея:** Алиса произносит имя звонящего: «Пришёл Иван Иванов».  
**Статус:** ❌ Отложено. Официальный API Яндекса не поддерживает передачу динамического текста. Неофициальные пути нестабильны.

### 3.2. Паттерн Gateway для внешних сервисов
**Идея:** Плагин регистрирует виртуальные датчики в ядре (например, для Яндекс Умный Дом), а ядро берёт на себя OAuth и Device Discovery.  
**Статус:** 🔄 В планах.

### 3.3. Автосканирование `/plugins`
**Текущее состояние:** Плагины загружаются явно в `PluginManager.loadPlugins()`.  
**Цель:** Автоматическое обнаружение любой папки с `index.ts/js` в `/plugins`.  
**Статус:** 🔄 В планах.
