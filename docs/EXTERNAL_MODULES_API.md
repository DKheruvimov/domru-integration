# API Внешних Модулей (External Modules API)

Внешние модули — это сторонние скрипты или приложения (например, Telegram-боты, n8n, Home Assistant), которые могут взаимодействовать с Ядром (Dom.ru Integration Hub) для получения событий с домофона и отправки команд (например, открытия двери).

## Авторизация (Токен)
Для работы с API вашему модулю нужен **Токен**. 
Токен можно сгенерировать в веб-интерфейсе Ядра (раздел "Настройки" -> "Внешние модули"). 

Авторизация передается в каждом HTTP-запросе одним из способов:
1. Через заголовок: Authorization: Bearer <ваш_токен> (рекомендуется)
2. Через query-параметр: ?token=<ваш_токен>

---

## 1. Получение событий (Webhook / WebSocket)

Модуль может получать события от Ядра (например, когда кто-то звонит в домофон) двумя способами: через Webhooks или через WebSockets. Модуль сам решает, какой способ ему удобнее.

### Способ А: Настройка Webhook (HTTP POST)
Если вы хотите, чтобы Ядро отправляло POST-запросы на ваш сервер при наступлении событий, вам нужно зарегистрировать свой Webhook URL в Ядре.

**Запрос (от Модуля к Ядру):**
POST /api/modules/me/webhook

**Заголовки:**
Authorization: Bearer <ваш_токен>
Content-Type: application/json

**Тело:**
`json
{
  "webhookUrl": "https://ваш-домен.com/api/webhook"
}
`

**Формат события (от Ядра к Модулю):**
Ядро будет отправлять POST-запросы на ваш webhookUrl с таким телом:
`json
{
  "event": "incoming_call",
  "payload": {
    "callId": "...",
    "intercomId": "...",
    "timestamp": 1690000000000
  }
}
`

### Способ Б: Подключение по WebSocket
Если вашему скрипту проще держать постоянное соединение (например, локальные Python-скрипты), вы можете подключиться по протоколу WebSocket.

**URL для подключения:**
ws://<адрес_ядра>/api/modules/ws?token=<ваш_токен>

После успешного подключения Ядро начнет транслировать все события домофона в виде JSON-сообщений.

---

## 2. Действия и Команды

Модуль может отдавать команды Ядру, например, открыть дверь домофона.

### Открыть дверь (Action Open)
**Запрос:**
POST /api/modules/actions/open

В заголовке передаем Authorization: Bearer <ваш_токен>.
В случае успеха вернется JSON { "success": true }.

---

## 3. Schema-Driven UI (Настройки Модуля)

Внешние модули могут динамически запрашивать у пользователя настройки через UI Ядра.
Для этого модуль должен зарегистрировать свою Схему Настроек. Ядро нарисует по этой схеме форму, и пользователь введет туда данные.

### Регистрация Схемы
**Запрос:**
POST /api/modules/me/schema

В заголовке: Authorization: Bearer <ваш_токен>

**Тело:**
`json
{
  "instruction": "Для работы бота получите токен у @BotFather в Telegram и вставьте его ниже.",
  "fields": [
    {
      "key": "bot_token",
      "type": "password",
      "label": "Токен бота",
      "required": true,
      "description": "Пример: 123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11"
    }
  ]
}
`

### Получение сохраненных настроек
Чтобы получить значения, которые пользователь ввел в интерфейсе Ядра, выполните запрос:

**Запрос:**
GET /api/modules/me/settings

В заголовке: Authorization: Bearer <ваш_токен>

В случае успеха Ядро вернет JSON-объект в формате ключ-значение:
`json
{
  "bot_token": "введенный_пользователем_токен"
}
`

## 4. Статусы и Мониторинг (UI State)

Ядро не "пингует" Webhook-модули и не пытается угадать их статус. Модуль **должен сам сообщать Ядру о своем состоянии**. Это состояние (и любые ошибки) мгновенно отображается пользователю в веб-интерфейсе.

### Отправка статуса
**Запрос:**
POST /api/modules/me/status

В заголовке: Authorization: Bearer <ваш_токен>
Content-Type: application/json

**Тело:**
`json
{
  "status": "online", 
  "message": "Модуль готов к работе! Токен Telegram проверен."
}
`

**Доступные статусы (status):**
- online — модуль работает исправно. В UI отображается зеленая галочка "В сети".
- offline — модуль выключен или не настроен. Серая вилка "Не в сети".
- warning — модуль требует внимания пользователя (например, нужно ввести настройки в интерфейсе). Желтый значок "Требуется настройка".
- error — произошла критическая ошибка в логике модуля (например, API Telegram недоступно). Красный значок "Ошибка".

*Параметр message выводится во всплывающей подсказке (tooltip) при наведении мышки на статус модуля в интерфейсе Ядра.*
