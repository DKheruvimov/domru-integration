## 📌 О проекте

Этот проект представляет собой полноценное решение для интеграции и управления устройствами умного дома от Дом.ru (домофоны, калитки, видеокамеры). Проект состоит из двух основных частей:

1. **Современная Веб-панель (React Dashboard):** Удобный, адаптивный интерфейс для просмотра камер, открытия дверей, управления гостевыми PIN-кодами и просмотра истории событий.
2. **TypeScript API Клиент (`domru-js`):** Мощная, типизированная библиотека для программного взаимодействия с экосистемой Дом.ru. Позволяет легко встраивать управление устройствами в ваши собственные скрипты или системы умного дома (например, Home Assistant).

> [!IMPORTANT]
> Проект не аффилирован с провайдером Дом.ru. Используйте его на свой страх и риск. Храните учётные данные только в безопасном окружении.

## ✨ Возможности

### 🖥️ Веб-интерфейс
- **Просмотр видео:** Интеграция с камерами домофонов в реальном времени (поддержка HLS/WebRTC через go2rtc).
- **Интеграция с Алисой:** Встроенная поддержка Yandex Smart Home (открытие дверей как замков) и Yandex Dialogs. Позволяет не только сказать *"Алиса, открой дверь"*, но и настроить умные сценарии вроде *"Алиса, жду курьера"* для запуска таймера автооткрытия!
- **Управление доступом:** Удалённое открытие дверей и калиток в один клик.
- **История событий:** Просмотр журнала звонков, снимков с камер при вызове и других уведомлений.
- **Гостевые коды и Расписания:** Генерация временных PIN-кодов для курьеров и гостей, а также создание гибких недельных расписаний автооткрытия для регулярных посетителей (клининг, няня, курьеры).
- **Синхронизация в реальном времени (WebSocket):** Мгновенная обратная связь в интерфейсе через WebSocket (Socket.IO). При включении, отключении или удалении карточки гостя/курьера автоматически синхронизируется статус автооткрытия на соответствующих камерах без необходимости обновлять страницу.
- **Умное управление временем:** Серверные проверки расписания автоматически нормализуются под Московский часовой пояс (MSK, UTC+3) независимо от региона размещения сервера. Просроченные временные пропуска автоматически вычищаются бэкендом.
- **Адаптивность и Дизайн:** Поддержка тёмной и светлой темы, мобильная и десктопная версии, всплывающие окна и "Личный кабинет". Применяются полностью кастомные модальные окна, совместимые с ограничениями песочницы iFrame (без использования блокирующих нативных `confirm`/`alert`).
- **Режим Песочницы (Demo):** Возможность протестировать интерфейс без реальных учётных данных.
- **Безопасность API:** Встроенные проверки `requireDomruAuth` защищают приватные роуты от неавторизованного доступа и Authentication Bypass. Управление конфигурацией (IP, порты) вынесено в централизованный `config.ts`.

### ⚙️ API Клиент (`domru-js`)
Библиотека находится в директории `src/domru-js/` и предоставляет следующие возможности:
- Автоматическое получение и обновление токенов доступа.
- Кеширование запросов для снижения нагрузки на серверы Дом.ru.
- Встроенный `EventEmitter` для отслеживания изменения токенов и ошибок.
- Методы для получения списка мест, устройств, камер, открытия дверей, получения снапшотов (JPEG) и видеопотоков.

Подробная документация по API доступна в [src/domru-js/README.md](src/domru-js/README.md).

## 🚀 Установка и запуск

Проект построен на связке **Node.js + Express** (бэкенд/прокси) и **React + Vite** (фронтенд).

**Требования:**
- Node.js (рекомендуется v18+)

### Локальная разработка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/DKheruvimov/domru-integration.git
   cd domru-integration
   ```

2. Установите зависимости:
   ```bash
   npm install
   ```

3. Запустите проект в режиме разработчика (Vite dev server + Node.js API):
   ```bash
   npm run dev
   ```
   *Приложение будет доступно по адресу: http://localhost:5173*

### Сборка для Production

1. Выполните сборку клиентской части и сервера:
   ```bash
   npm run build
   ```
2. Запустите собранный сервер:
   ```bash
   npm start
   ```
   *Сервер запустится, используя параметры из `server/config.ts` (по умолчанию порт `3000` и хост `localhost`). Вы можете переопределить их через переменные окружения:*
   ```bash
   PORT=8080 HOST=0.0.0.0 npm start
   ```

> [!TIP]
> **Развертывание и Безопасность**
> Если вы разворачиваете приложение на публичном сервере (VPS) в РФ, настоятельно рекомендуется спрятать сервер за бесплатный **Reverse Proxy (например, TurboFlare или Cloudflare)** в режиме `Full Strict`. Это защитит ваш сервер от прямых DDoS-атак, сканеров портов и скачков входящего IP, на которые реагирует блокировками ваш облачный провайдер. В проекте встроен `app.set("trust proxy", true)`, поэтому реальные IP клиентов будут определяться корректно.

## 📖 Краткий пример использования API (`domru-js`)

Если вы хотите использовать только API-клиент в своём Node.js проекте:

```typescript
import { DomruClient } from "./src/domru-js";

const client = new DomruClient({
  login: "ваш_логин",
  password: "ваш_пароль"
});

await client.authenticate();

const places = await client.getSubscriberPlaces();
if (places.length > 0) {
  const placeId = places[0].place.id;
  const devices = await client.getDevices(placeId);
  
  // Открыть первую доступную дверь
  if (devices[0]) {
    const result = await client.openDoor(placeId, devices[0].id);
    console.log("Дверь открыта:", result.status);
  }
}
```
