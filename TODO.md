# Задачи на следующую сессию

## 1. Архитектурный рефакторинг: Перенос WebSockets и API на сабдомен
**Цель:** Обойти блокировки Anti-DDoS провайдеров (TurboFlare / Cloud.ru), которые режут заголовки `Upgrade` (ломая WebSockets) и банят IP за HTTP-polling.

**Шаги для реализации:**
1. **Настройка DNS:** 
   - Создать в DNS-панели новую A-запись для сабдомена (например, `api.kheruvimov.ru` или `ws.kheruvimov.ru`).
   - Направить эту запись **напрямую** на IP-адрес сервера (176.123.161.43), минуя проксирование CDN (облачко должно быть серым, без TurboFlare/Cloudflare).
2. **Настройка Nginx на сервере:**
   - Добавить новый `server` блок для сабдомена `api.kheruvimov.ru` в `nginx.conf`, который будет проксировать трафик на Node.js (`http://127.0.0.1:3100`), с обязательной поддержкой заголовков `Upgrade` и `Connection "upgrade"`.
   - Получить SSL-сертификат для нового сабдомена (через `certbot`).
3. **Обновление фронтенда (`src/socket.ts`):**
   - Изменить URL подключения Socket.io с относительного `/` на абсолютный `wss://api.kheruvimov.ru`.
   - Вернуть форсированное использование WebSockets (`transports: ["websocket"]`), так как при прямом подключении они будут работать стабильно.
4. **Опционально (Обновление API):**
   - Перевести все `fetch` запросы фронтенда на новый `api.kheruvimov.ru`, чтобы вообще исключить ложные срабатывания WAF при массовой загрузке картинок камер (`/api/domru/snapshot/...`).
