# Задачи на следующую сессию

## ~~1. Архитектурный рефакторинг: Перенос WebSockets и API на сабдомен~~ (ВЫПОЛНЕНО)
**Цель:** Обойти блокировки Anti-DDoS провайдеров (TurboFlare / Cloud.ru), которые режут заголовки `Upgrade` (ломая WebSockets) и банят IP за HTTP-polling.

**Результат:**
- Создан сабдомен `api.kheruvimov.ru`, настроен `nginx` с поддержкой `Upgrade`.
- С WebSockets снято проксирование TurboFlare, они работают напрямую и стабильно.
- *ВАЖНО:* Пункт с переключением всех `fetch` запросов на `api.kheruvimov.ru` был отменён. Кросс-доменные запросы порождали шквал `OPTIONS` preflight-ов, что приводило к мгновенному бану IP со стороны Cloud.ru Anti-DDoS WAF. Запросы к API оставлены на `kheruvimov.ru` (через TurboFlare).

## ~~2. Разработка внешнего модуля Face ID~~ (БАЗА ВЫПОЛНЕНА)
**Цель:** Реализовать микросервис распознавания лиц на Python и интегрировать его с Ядром `domru-integration`.

**Результат:**
- Создан модуль в папке `face-id-module`.
- Реализовано WebSocket-соединение с Ядром (с обходом WAF через User-Agent и правильный namespace `/modules`).
- Модуль успешно регистрирует `FACE_RECOGNITION` Capability, скачивает фото людей из Ядра и захватывает RTSP/HLS поток при входящем звонке.

## 3. Задачи на следующую сессию: Тестирование Face ID
**Цель:** Провести полноценное (E2E) тестирование модуля распознавания лиц.

**План действий (Шаги для проверки):**
1. В интерфейсе домофона (вкладка «Люди») включить тумблер `Face ID` у любого пользователя.
2. Загрузить фотографию пользователя в хранилище (через интерфейс).
3. Перезапустить Python-модуль (`uvicorn src.face_service.main:app`), чтобы он скачал загруженную фотографию и построил эмбеддинг.
4. Сымитировать входящий звонок с панели домофона (или дождаться реального звонка).
5. Проверить в терминале Python-модуля, что захват видеокадров начался.
6. Убедиться, что лицо корректно распознаётся, и модуль отправляет команду открытия двери (`POST /api/modules/actions/open`).
7. Проверить логи Ядра, чтобы убедиться, что дверь фактически открылась и событие зафиксировалось.

**Дополнительные доработки модуля (По необходимости):**
- Автоматическая перезагрузка фото (без перезапуска скрипта) при получении события об обновлении хранилища от Ядра.
- Оптимизация детекции (FPS, размер буфера кадров OpenCV).
