# Dom.ru Integration - AI Rules & Guidelines

Этот файл содержит специфичные инструкции для ИИ-агентов (Antigravity и др.), работающих с кодовой базой проекта. Пожалуйста, внимательно ознакомьтесь с этими правилами перед началом любых изменений.

## 1. Архитектура и Модули (Plugins & Modules)
Проект трансформируется в архитектуру **Smart Intercom Hub**.
- Подробная документация по архитектуре плагинов и модулей находится в `docs/PLUGINS_ARCHITECTURE.md`. Обязательно прочитайте этот документ перед реализацией новых функций интеграции.
- **External Modules (Внешние модули):** Подсистема УЖЕ РЕАЛИЗОВАНА. Не ломайте обратную совместимость WebSocket пространства `/modules` и REST API `/api/modules/actions/open`. Настройки хранятся в `data/modules.json`.
- **Internal Plugins (Внутренние плагины):** Если пользователь просит вас реализовать внутренние плагины:
  - Плагины должны размещаться в папке `plugins/`.
  - Конфигурации плагинов (настройки, токены) категорически ЗАПРЕЩЕНО хранить в `.env` файлах. Используйте централизованное хранилище (например, `data/plugins.json`), управляемое Ядром.
  - UI для плагинов должен генерироваться Ядром динамически (на основе схемы, экспортируемой плагином).

## 2. Пользовательский интерфейс и Дизайн (КРИТИЧНО)
- **Строгий стиль:** Строго соблюдайте существующий визуальный стиль React-приложения. Пользователь прямо запретил "цветовую мешанину".
- **Стилизация:** Используйте только те классы TailwindCSS, которые уже применяются в проекте:
  - Фоны: `bg-white`, `bg-zinc-50`, `bg-zinc-900`, `bg-[#0B0F12]`, `bg-[#161b22]`.
  - Тексты: `text-zinc-900`, `text-zinc-500`, `text-zinc-400`.
  - Акценты: `text-emerald-500` (для статусов успеха/онлайн), `bg-[#E30613]` (фирменный красный Дом.ru).
  - Элементы: Карточки с закруглениями `rounded-2xl` или `rounded-xl`, обводки `border-zinc-200 / border-zinc-800`.
- **Иконки:** Используйте только библиотеку `lucide-react`.

## 3. Стек технологий
- **Backend:** Node.js (Express), WebSockets (Socket.io). Код в директории `server/`.
- **Frontend:** React, Vite, TailwindCSS. Код в директории `src/`.
- Сборка и запуск: Код собирается через `npm run build` (Vite для фронтенда и ESBuild для бэкенда). Не используйте TypeScript компилятор напрямую для продакшн сборки. 

## 4. Общие правила
- Не переписывайте существующую логику перехвата SIP (SIP interception) без явного указания.
- При создании новых файлов и маршрутов (Express routes), не забывайте подключать их в `server.ts`.
- Сопровождайте крупные архитектурные изменения обновлением документации.
